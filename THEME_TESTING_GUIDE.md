# 主题设置功能测试指南

## ✅ 已完成

AppearanceSetting 组件已成功集成到设置页面中，现在可以通过界面设置深色/浅色主题了。

## 🧪 测试步骤

### 1. 启动应用
```bash
pnpm run dev:ui
# 或
pnpm run tauri dev
```

### 2. 打开设置页面
- 在主界面点击设置图标
- 或使用快捷键打开设置

### 3. 切换到"外观"标签页
- 在左侧导航栏点击"外观"（调色板图标）

### 4. 测试主题切换功能

#### 4.1 手动切换主题
- **深色主题按钮**：点击后界面应变为深色模式
  - 应该看到深色背景、浅色文字
  - 按钮上应显示蓝色选中标记 ✓
  
- **浅色主题按钮**：点击后界面应变为浅色模式
  - 应该看到浅色背景、深色文字
  - 按钮上应显示蓝色选中标记 ✓

#### 4.2 跟随系统开关
- 打开"跟随系统主题"开关
  - 主题应立即切换为当前系统主题
  - 深色/浅色按钮的选中状态应更新
  
- 关闭"跟随系统主题"开关
  - 可以再次手动选择主题

#### 4.3 测试系统主题变化
1. 打开"跟随系统主题"开关
2. 更改操作系统的主题设置
   - Windows: 设置 > 个性化 > 颜色 > 选择模式
3. 观察应用是否自动切换主题

### 5. 测试其他外观设置

#### 5.1 动画效果
- 打开/关闭"启用动画效果"开关
- 观察界面过渡动画是否启用/禁用

#### 5.2 布局密度
- 选择"紧凑"模式：搜索结果列表更密集
- 选择"宽松"模式：搜索结果列表更舒适

#### 5.3 窗口透明度
- 拖动滑块调整透明度（50% - 100%）
- 观察窗口背景透明度的变化

## 🐛 常见问题排查

### 问题 1: 点击按钮没有反应
**可能原因**: 
- ThemeProvider 未正确包裹应用
- next-themes 未正确初始化

**解决方案**:
```tsx
// 检查 main.tsx
<ThemeProvider>
  <App />
</ThemeProvider>
```

### 问题 2: 主题切换但样式不变
**可能原因**:
- Tailwind CSS dark mode 配置问题
- HTML class 未正确更新

**解决方案**:
```tsx
// 检查 ThemeProvider 配置
<NextThemesProvider attribute="class" defaultTheme="light">
```

### 问题 3: 刷新后主题重置
**可能原因**:
- localStorage 未正确保存
- AppSettings 同步失败

**解决方案**:
- 检查浏览器控制台 localStorage
- 确认 updateSettings 被调用

### 问题 4: 跟随系统不工作
**可能原因**:
- matchMedia 监听器未注册
- syncWithSystemTheme 状态未更新

**解决方案**:
- 检查 ThemeProvider 中的 useEffect
- 确认事件监听器正确添加

## 📊 调试技巧

### 查看当前主题
```javascript
// 在浏览器控制台中
document.documentElement.classList
// 应该包含 'dark' 或 'light'
```

### 查看设置状态
```javascript
// 在浏览器控制台中
JSON.parse(localStorage.getItem('app-settings'))
// 检查 theme 和 syncWithSystemTheme 字段
```

### 手动触发主题切换
```javascript
// 在浏览器控制台中
import { useTheme } from 'next-themes'
// 或在 React DevTools 中检查 useTheme hook
```
