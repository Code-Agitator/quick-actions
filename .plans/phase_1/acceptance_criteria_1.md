# 模块 1: 外观增强设置 - 验收标准

## 功能完整性

- [ ] **动画开关**：Switch 组件正确显示 `settings.enableAnimations` 的状态
- [ ] **动画开关**：切换 Switch 后，页面过渡动画立即启用/禁用
- [ ] **布局密度选择**：Select 组件显示当前密度选项（compact/comfortable）
- [ ] **布局密度选择**：切换密度后，`settings.layoutDensity` 更新并持久化
- [ ] **窗口透明度滑块**：Slider 组件显示当前透明度值（50%-100%）
- [ ] **窗口透明度滑块**：拖动滑块时，窗口透明度平滑变化
- [ ] **窗口透明度滑块**：松开滑块后，设置自动保存（带防抖）

## 命名一致性

- [ ] 所有变量名符合 camelCase 规范（如 `windowOpacity`、`layoutDensity`）
- [ ] 组件名符合 PascalCase 规范（如 `AppearanceSetting`）
- [ ] 布尔状态以 `is`、`has`、`can` 开头（如 `enableAnimations`）
- [ ] 事件处理函数以 `handle` 开头（如有）

## 规范遵循

- [ ] 代码风格符合 project-conventions.md
- [ ] TypeScript 类型安全，无 `any` 类型使用
- [ ] 所有异步操作包含错误处理（如有）
- [ ] 无空的 catch 块

## 编译/运行检查

- [ ] TypeScript 编译无错误
- [ ] ESLint 检查无警告
- [ ] 应用成功启动无崩溃
- [ ] AppearanceSetting 组件正常渲染

## UI/UX 验证

- [ ] 三个新设置项在"外观"标签页中正确显示
- [ ] 每个设置项都有清晰的图标、标题和描述
- [ ] 设置项之间的间距和布局一致
- [ ] 暗色主题下颜色和对比度适配良好
- [ ] 响应式布局在不同屏幕尺寸下正常显示

## 功能测试

### 动画开关测试
- [ ] 默认状态为开启（true）
- [ ] 关闭后，页面元素过渡动画消失
- [ ] 重新开启后，动画恢复
- [ ] 刷新页面后状态保持

### 布局密度测试
- [ ] 默认状态为 "comfortable"
- [ ] 切换到 "compact" 后，localStorage 更新
- [ ] 刷新页面后选项保持
- [ ] （如果CSS已实现）界面元素间距变化明显

### 透明度滑块测试
- [ ] 默认值为 0.98（98%）
- [ ] 拖动到最小值 0.5，窗口半透明
- [ ] 拖动到最大值 1.0，窗口完全不透明
- [ ] 快速拖动时，不会频繁更新（防抖生效）
- [ ] 刷新页面后值保持

## 性能要求

- [ ] 透明度滑块拖动时，帧率保持在 50fps 以上
- [ ] 切换设置时，无明显卡顿或延迟
- [ ] 内存占用增加不超过 5MB

## 边界条件

- [ ] 透明度值不会低于 0.5 或高于 1.0
- [ ] 布局密度只能是 "compact" 或 "comfortable"
- [ ] 动画开关只能是 true 或 false
- [ ] 异常输入有合理的默认值

## 代码质量

- [ ] 新增代码行数不超过 150 行
- [ ] 单个函数长度不超过 50 行
- [ ] 组件文件总长度不超过 350 行
- [ ] 有适当的注释说明复杂逻辑
- [ ] 无重复代码

## 集成测试

- [ ] 与其他设置项（主题、语言）不冲突
- [ ] ThemeProvider 正常工作
- [ ] localStorage 读写正常
- [ ] 无控制台错误或警告
