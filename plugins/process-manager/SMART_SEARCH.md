# 智能搜索功能说明

## 🎯 核心理念

**一个搜索框，全能搜索！** 

无需手动切换搜索模式，系统会自动识别你的输入类型。

---

## 🧠 智能识别规则

### 1. 🔌 端口号识别

**触发条件**: 纯数字且在 1-65535 范围内

**示例**:
```
输入: 8080
识别: 端口搜索
执行: 查找占用 8080 端口的进程
```

**典型场景**:
- `80` - HTTP 默认端口
- `443` - HTTPS 默认端口
- `3306` - MySQL 数据库
- `8080` - Tomcat/Node.js 开发服务器
- `5432` - PostgreSQL 数据库

### 2. 📁 文件路径识别

**触发条件**: 包含以下任一特征
- Windows 盘符（如 `C:\`）
- 正斜杠 `/`
- 反斜杠 `\`

**示例**:
```
输入: C:\Users\test\document.docx
识别: 文件搜索
执行: 查找占用该文件的进程

输入: D:/projects/app/index.js
识别: 文件搜索
执行: 查找占用该文件的进程
```

**典型场景**:
- 文件被锁定无法删除
- 想知道哪个程序在使用某个文件
- 排查文件占用问题

### 3. 📝 名称/PID 搜索

**触发条件**: 不满足上述两种条件（默认）

**示例**:
```
输入: chrome
识别: 名称搜索
执行: 查找名称包含 "chrome" 的进程

输入: 1234
识别: 名称搜索（同时匹配PID）
执行: 查找 PID 为 1234 或名称包含 "1234" 的进程
```

**支持**:
- 进程名称模糊匹配
- PID 精确匹配
- 大小写不敏感

```
用户输入: chrome
↓
系统识别: 📝 进程名称
↓

```

```
↓
```

## 🔍 识别优先级

系统按以下顺序判断：

```
1. 是否是有效端口号？ (1-65535 的纯数字)
   ↓ 是 → 端口搜索
   
2. 是否包含文件路径特征？ (盘符、斜杠)
   ↓ 是 → 文件搜索
   
3. 默认
   ↓ → 名称/PID 搜索
```

## 🔧 技术实现

### 检测函数

```typescript
const detectSearchType = (query: string): { 
  type: 'port' | 'file' | 'name'; 
  value: string 
} => {
  const trimmed = query.trim();
  
  // 1. 检查是否是纯数字（端口号）
  if (/^\d+$/.test(trimmed)) {
    const port = parseInt(trimmed);
    if (port >= 1 && port <= 65535) {
      return { type: 'port', value: trimmed };
    }
  }
  
  // 2. 检查是否是文件路径
  if (/^[a-zA-Z]:\\/.test(trimmed) || 
      trimmed.includes('/') || 
      trimmed.includes('\\')) {
    return { type: 'file', value: trimmed };
  }
  
  // 3. 默认按名称搜索
  return { type: 'name', value: trimmed };
};
```

### 搜索执行


## 💬 常见问题

### Q1: 为什么输入 "1234" 没有识别为端口？

**A**: 如果 1234 不是有效的端口号（超出 1-65535 范围），会被识别为名称搜索。但在这个例子中，1234 是有效端口，所以会被识别为端口搜索。如果你想搜索名称包含 "1234" 的进程，可以添加字母，如 "app1234"。

### Q2: 文件路径识别不准确怎么办？

**A**: 确保路径格式正确：
- Windows: `C:\path\to\file` 或 `C:/path/to/file`
- 包含盘符或斜杠才会被识别为文件路径

### Q3: 可以强制指定搜索类型吗？

**A**: 当前版本不支持，但不需要。智能识别已经非常准确。如果有特殊需求，可以通过添加特定前缀来实现（未来可能支持）。

### Q4: 搜索速度慢怎么办？

**A**: 
- 名称搜索是即时的（本地过滤）
- 端口/文件搜索需要调用 API，可能需要几百毫秒
- 这是正常的，系统会显示 loading 状态
