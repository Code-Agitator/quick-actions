## 2. 代码质量规范

### 2.1 错误处理
- 所有异步函数必须包含try-catch或.catch()
- 禁止空的catch块，必须记录日志或重新抛出
- Rust代码中使用Result类型处理错误，优先使用?运算符传播错误
- 禁止使用unwrap()，必须提供有意义的错误信息
- 资源清理失败时应记录警告，但不中断主流程

### 2.2 文件安全
- 处理用户上传的ZIP/TAR等压缩文件时，必须进行安全验证
- 拒绝包含".."的路径（防止路径遍历攻击）
- 拒绝绝对路径（以/或\开头的路径）
- 验证完成后才能解压文件到目标目录
- 解压失败时必须清理已创建的文件（失败回滚）

### 2.3 资源管理
- 使用RAII模式管理临时资源（如TempDir自动清理）
- 文件操作失败时应尝试清理已创建的资源
- 清理失败应记录警告信息，包含具体错误原因
- 避免静默失败，关键操作的结果必须反馈给用户

### 2.4 用户交互规范
- 禁止使用 `alert()` 和 `confirm()` 原生对话框
- 应使用 Toast、Modal 或自定义对话框组件提供用户反馈
- 错误提示应友好且包含可操作的解决方案
- 成功操作应有明确的视觉反馈（如Toast、动画）

### 2.5 TypeScript 类型安全
- 禁止使用 `as any` 类型断言绕过类型检查
- 应定义明确的接口类型或使用类型守卫（type guards）
- 对于外部API返回的数据，必须进行运行时类型验证
- 优先使用 `unknown` 而非 `any`，并通过类型收窄确保安全

## 5. 插件开发规范

### 5.1 插件打包
- 每个插件必须支持 `pnpm run pack` 命令生成 ZIP 包
- ZIP 文件命名格式：`{plugin-id}-{version}.zip`
- ZIP 内容必须包含：dist/, plugin.json, README.md（如果存在）
- 使用 archiver 库进行打包，压缩级别为 9（最高）
- 打包前必须先执行 `pnpm build`
- 打包脚本位于 `scripts/pack-plugin.js`，所有函数必须有完整的 JSDoc 注释包括 `@throws` 标签
- ZipArchive 构造函数只接受一个 options 参数，正确用法：`new ZipArchive({ zlib: { level: 9 } })`
